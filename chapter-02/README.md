**第二章：路由（Routing）**
作者把它定义成：当系统不再只是按固定顺序执行，而是需要**根据输入、环境状态或前一步结果，动态决定“下一步走哪条路”**时，就需要路由模式。([jimmysong.io][1])

先抓这一章最重要的一句话：

**提示链**解决的是“把复杂任务拆成顺序步骤”；
**路由**解决的是“面对不同情况，应该进入哪条不同的处理路径”。([Jimmy Song][1])

你可以把它想成一个“分诊台”。

比如一个客服智能体先判断用户意图：
用户问订单，就走订单查询链；
用户问产品，就走产品检索链；
用户问技术问题，就走故障排查或转人工链；
意图不清楚，就先进入澄清问题链。书里就是这样举例说明的。([jimmysong.io][1])

---

## 这一章在讲什么

路由的本质，是给智能体加入**条件逻辑**。
这样系统就不再只有一条固定执行路径，而是能从多个候选动作、工具或子流程中，选择最合适的那个。作者强调，这会让系统从“静态流程”变成“上下文感知的动态工作流”。([jimmysong.io][1])

书里提到 4 种典型实现方式：

1. **基于 LLM 的路由**
   让模型先做分类，再输出路由结果。
   例如提示它：“只输出类别：订单状态 / 产品信息 / 技术支持 / 其他”。([jimmysong.io][1])

2. **基于嵌入的路由**
   把用户输入转成向量，再和各个路由目标的向量做相似度比较，选最接近的一条路径。适合“按语义分流”。([jimmysong.io][1])

3. **基于规则的路由**
   用 if-else、关键词、正则、结构化字段来做判断。速度快、确定性强，但灵活性差。([jimmysong.io][1])

4. **基于机器学习分类器的路由**
   不是用大模型现场判断，而是训练一个专门的分类模型负责分流。适合高频、稳定、可标注的数据场景。([jimmysong.io][1])

## 怎么理解“什么时候该用路由”

书里给了一个很好的经验法则：
**当智能体需要在多个工作流、工具或子智能体之间做选择时，就该用路由。**
典型场景就是客服、虚拟助手、AI 教师、工单分发、邮件分类、文档处理流程等。([Jimmy Song][1])

换成更接地气的话：

* 只有一条固定流程，用提示链就够了
* 出现“岔路口”，就需要路由
* 岔路口越多，路由越重要

## 这章和你现在做知识库助手有什么关系

非常大。

你前面在讨论的那个知识库助手，其实天然就会碰到路由问题。比如同一个用户提问进来后，系统可能要先判断：

* 这是在问**产品使用说明**
* 还是在问**开放平台 API 文档**
* 还是在问**文档检索**
* 还是在问**需要转人工 / 澄清补充信息**

这就是一个非常典型的路由层。
也就是说，你们不是“拿到问题直接搜”，而是常常先要有一个**router**，决定后面应该走哪种处理链。这个思路和本章完全一致。([Jimmy Song][1])

## 这一章的关键收获

作者在“关键要点”里总结得很直接：

* 路由让智能体能根据条件动态决定下一步
* 它让系统突破线性执行的限制
* 路由可以用 LLM、规则、嵌入相似度来实现
* LangGraph、Google ADK 等框架都支持结构化路由，但风格不同 ([Jimmy Song][1])

其中有一个特别值得你记住：

**路由不是附属技巧，而是让智能体真正变“智能”的关键一步。**
因为真实世界的问题本来就不是单线路径。作者在总结里也明确说，路由是构建“动态、响应式智能体系统”的关键。([Jimmy Song][1])

## 我帮你翻译成工程语言

你可以把第二章浓缩成一个工程模板：

```text
用户输入
  ↓
路由器（Router）
  ├─ FAQ / 文档问答链
  ├─ RAG 检索链
  ├─ 工具调用链
  ├─ 转人工链
  └─ 澄清问题链
```

这就是路由模式的核心结构。
它本质上是一个“总控节点”，决定后续要走哪个子系统。

## 你最该警惕的一个误区

很多人一开始做 Agent，会把所有能力都塞进一个大 prompt，让模型“自己看着办”。
第二章其实是在告诉你：**更好的方式是先分流，再让专门模块处理。**

也就是：

* 不要一个万能链做所有事
* 要把任务拆成多个专用能力
* 用路由决定把请求送到哪里

这会让系统更稳定、更可测试，也更容易迭代。这个结论虽然是从章节内容推出来的工程理解，但和作者对“动态选择工具、函数或子智能体”的描述是一致的。([Jimmy Song][1])

## 这一章读完后，你应该能回答 3 个问题

1. **什么是路由？**
   就是根据条件动态选择下一步流程。([jimmysong.io][1])

2. **为什么需要路由？**
   因为真实任务有分支，线性链条不够。([Jimmy Song][1])

3. **怎么实现路由？**
   LLM、规则、嵌入、分类模型都可以。([jimmysong.io][1])

## 第二章一句话总结

**路由模式 = 让智能体先判断，再决定走哪条处理路径。**
它是从“固定流程”迈向“自适应系统”的第一步。([Jimmy Song][1])
